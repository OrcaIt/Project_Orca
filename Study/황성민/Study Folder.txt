클라우드 : 각각 고유의 기능을 가진 서버의 글로벌 네트워크

클라우드 컴퓨팅 : IT 리소스를 인터넷을 통해 온디멘드로 제공하고 사용한 만큼만 비용을 지불하는 것
- 클라우드에서 서버, 스토리지, 소프트웨어 등 필요한 IT 자원을 받아서 사용하는 기술
- 인터넷 상의 서버에 단순히 자료를 저장하는 것 뿐만 아니라 별도의 프로그램 설치없이 웹에서 제공하는 응용프로그램의 기능을
이용하여 원하는 작업을 수행, 여러사람이 동시에 문서를 공유하면서 작업을 진행할 수 있음

클라우드 서비스 이점
1) cost
- 쓴 만큼의 비용만 지불하면 된다.
2) 생산성
- 테스트해보고 싶으면 짧은시간에 환경을 구축할 수 있음. 시간단축으로 업무에 집중가능
3) 진입장벽
- 신기술에 대한 진입장벽을 낮출 수 있음. 클라우드를 제공하는 업체에서 솔루션,패키지화해서 제공하고 있기 때문에 쉽게 테스트 가능
  맞지 않으면 빨리 변경할 수 있고, 시작할 경우에도 환경구축이 빨리 가능

클라우드 서비스 단점
1)엄청난 인터넷 전송 용량에 따른 초과 비용의 위험
2) App 퍼포먼스의 문제(네트워크 지연 고려)
3) 클라우드에 적합하지 않은 데이터
4) 회사 규모에 따른 제한
5) 인적 자원의 부재

클라우드 특징
1) 내결함성(Fault tolerance) : 운영중이던 시스템의 데이터가 손실되거나 진행중인 작업이 손생되지 않도록 돌발에 대비할 수 있음
2) 고 가용성(High availability) : 시스템이 오랜기간동안 지속적으로 정상운영되도록 함.(고장나지않음)
3) 재해 복구(Disaster recovery) : 1, 2번에 포함됨
4) 확장성(Scalability) : 온-프레미스서버라면 서버를 확장하는데 물리적으로 시간이 많이 필요함. 단, 클라우드는 원할때 바로 확장 가능(scale up, scale out 등 필요할경우 바로 늘려 고객 만족도 상승가능)
5) 민첩성(Agility) : 확장을 원하는 순간 빠른 시간에 서버를 배포, 서비스를 지속적으로 이어갈 수 있음
6) 탄력성(Elasticity) : 필요할 때 언제든지 늘릴 수 있고, 필요하지 않을때 언제든지 줄일 수 있다.(확장성과 다른점 : 줄어드는것에도 초점을 둠)
7) 글로벌 지원(Global reach) : 기본적으로 글로벌 지원 가능. 고객이 그들의 워크로드를 올려서 사용할 때, 그들의 고객(엔드유저)에게 보다 빠른속도로 서비스 제공하기 위해 지원
8) 응답 속도(Customer latency) : 리전에 데이터센터를 두고 빠른 응답속도를 지원
9) 예측 비용(Predicive cost) : 쓴만큼 비용을 지불하기 때문에 비용을 정확하게 산정할 수 있어 비용예측에 장점이 있음
10) 보안(Security) : 암호화, 감사 등의 기술적인 부분 외에도, 고객의 워크로드를 올려 서비스 해야하기 때문에, 지역적, 산업적인 준수사항 및 요구사항을 만족하도록 구현하고 검증받음.

클라우드 서비스 종류
- SaaS(Software as a Service)
- PaaS(Platform as a Service)
- IaaS(Infrastructure as a Service)
- Faas(Function as a Service)

1. SaaS(Software as a Service)
- 클라우드 서비스 중 가장 일반적, 누구나 한번쯤 사용해 본 경험이 있는 유형.
- 서비스 제공자가 모든 인프라와 소프트웨어 제품을 제공
- 사용자는 웹 기반의 프론트엔드를 사용하여 서비스와 상호하여 필요할 때 원하는 비용만 내면 어디서든 곧바로 사용가능
- PC나 기업 서버에 소프트웨어를 설치할 필요가 없는 만큼 S/W설치를 위해 시간,비용을 들이지 않아도 됨
- 중앙에서 S/W가 관리하기 때문에 사용자가 일일이 업그레이드나 패치할 필요 없음
- 대표 SaaS) 구글 앱스, 세일스포스 닷컴, MS오피스 365, 드롭박스 등 클라우드 스토리지 서비스

2. PaaS(Platform as a Service)
- 서비스 제공자가 자체 하드웨어 인프라에서 호스트하는 S/W와 제품개발도구를 제공하는 클라우드 서비스
- 개발기술을 모듈화, 표준화하는 능력, 기반에 최적화해 설치하는 기술력, 지원력 등의 기술요건이 필요
- 모니터링이나 프로비저닝, 비용처리방식(Billing)도 솔루션 개발사에 유리하게 구성
- 대표 PaaS) Google App Engine

3. IaaS(Infrastructure as a Service)
- 가상서버, 데이터 스토리지 및 DB와 같은 일련의 빌딩 블록에 대한 엑세스를 서비스 형태로 제공하는 클라우드 서비스
- 대규모 데이터 센터의 서버 구축 및 운용 능력, 자본력, 넓은 대지, 풍부한 전력환경 등이 필요
- 가상 공간의 서버 자원을 제공하는서비스, 서버의 확장이 가능. 윈도우/리눅스 모두 지원
- 사용자는 이러한 빌딩 블록을 결합하거나 계층 구조화하여 Application 실행하는데 필요한 인프라를 구축할 수 있음
- 대표 IaaS) AWS(Amazon Web Services), Rackspace

4. FaaS(Function as a Service)
- 대표 Faas) 람다함수

각 클라우드 서비스별 장단점
1)IaaS
장점 : 인프라에 대한 관리 및 제어를 최대 수준으로 제공. 확장성이 매우 뛰어나고 가장 유연한 클라우드 컴퓨팅 모델로 간주
        필요한 자원은 서비스로 제공되며 필요에 따라 구매하거나 소비별로 구매할 수 있으며, 흔히 종량제(pay as-you-use)모델로서 가격책정기준이 됨
        일반적으로 여러 사용자가 단일 하드웨어를 사용할 수 있도록 한다.
단점 : 보안 및 클라우드 인프라와 기존 애플리케이션 호환성에 대한 일반적인 우려사항.
        고객은 App, data, OS 및 미들웨어를 완벽하게 제어할 수 있으며, 이러한 App 중 어느하나라도 실패하면 프로세스와 운영에 영향을 미칠 수 있음
        -> App이 모두 안전하고 제대로 작동하는지 확인할 책임이 있음

2)PaaS
장점 : 개발자가 보안 패치 및 업데이트와 같은 주변적인 문제에 얽매이지 않고 고유한 맞춤형 App을 생성, 테스트 및 배포 등 당면한 작업에 집중할 수 있음
        가상화 기술을 중심으로 설계되었으며, 시스템 관리에 대한 배경이 없는 사람에게도 쉽고 간편하게 사용할 수 있도록 구축.
        확장성이 뛰어나 프로젝트 범위에 맞는 다양한 구성 요소 및 리소스 계층을 선택할 수 있음.
        ->시간과 비용절감을 위한 클라우드 컴퓨팅 솔루션으로 만듦
단점 : S/W와 App가 구축되는 플랫폼에 대한 제어력은 뛰어나나 플랫폼에 구축되는 플랫폼에만 제어가 제한되는점.
        플랫폼을 지원하는 OS나 하드웨어(및 벤더가 제공하는 다른 PaaS 구성요소)에 오작동이나 장애와 같은 문제가 발생할 경우 S/W와 데이터에 영향을 줄 수 있음

3)SaaS
장점 : 편의성, 경제성, 사용 편의성, 자동화, 다른 App과의 통합, 사용자 지정, 확장성
        강력한 기능과 기능을 제공하고 최신기술을 활용.
        SaaS도구는 또한 특정 비즈니스 기능에 초점을 맞추도록 설계됨
        모든 비즈니스 과제, 프로세스, 프로젝트, 워크플로우, 사무실 및 생산성 요구사항에 대한 SaaS솔루션이 있음
단점 : 고객이 완전히 통제 할 수 없다. S/W외관, 업데이트, 버전, 데이터, 거버넌스에 관한 한 SaaS는 벤더의 손에 맡겨지는 방식임
        - 워크플로우와 프로세스에 맞게 S/W 구성할 수 있지만 고객은 사용자-엔드 보안(user-end security)을 도입, SaaS 솔루션의 기능에 맞게 거버넌스 절차를 재정의 해야할 수도 있음
       보안에 대해 말하자면 특히 최근 몇 년 동안 대규모 데이터 침해와 보안 해킹에 비추어 클라우드에 중요한 데이터를 저장하는 기업들에게 큰 이슈가 되어 왔다. SaaS 우려 사항과 보안 위험에 대해 잘 알고 있으며
       신뢰할 수 있는 SaaS 제공업체를 사용하는 것이 중요하다. SaaS제공 업체를 이용하면 강력한 보안, 제로 다운타임 및 최적의 소프트웨어 성능뿐만 아니라 양질의 고객 지원과 지원을 보장받을 수 있다.
